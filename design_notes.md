# ゲームモード分離設計

## 1. モード定義
- **フリーモード (Free Mode)**
  - 全てのタグが最初から使用可能。
  - オンライン対戦、ローカル対戦が可能。
  - キャラクタービルド、パーティ編成が可能。
- **ストーリーモード (Story Mode)**
  - 初期タグ（6つ）から開始。
  - ステージクリアでタグを解放。
  - ストーリー専用のビルド・編成画面を持つ。
  - クリア状況、解放済みタグを保存。

## 2. データ構造 (LocalStorage)
- `tm_free_chars`: フリーモードで作成したキャラ
- `tm_free_parties`: フリーモードで作成したパーティ
- `tm_story_chars`: ストーリーモードで作成したキャラ
- `tm_story_parties`: ストーリーモードで作成したパーティ
- `tm_story_progress`: ストーリーの進行状況（クリア済みステージID、解放済みタグID）

## 3. UI構成
- **タイトル画面**: 「ストーリーモード」「フリーモード」の選択。
- **共通ナビゲーション**: 
  - ビルド / パーティ / (ストーリー) / 対戦 / タイトルへ戻る
- **モード切替**:
  - `currentMode` 変数で管理し、UIの表示内容（タグリスト等）を動的に変更。

## 4. タグ解放仕様
- 初期タグ: 101, 102, 103, 104, 105, 106 (仮)
- ストーリー報酬: `data.json` の `rewardTagId` に基づく。
